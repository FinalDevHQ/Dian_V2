# Frontend Conventions

## Tech Stack

- **Framework**: React 19 + TypeScript
- **Bundler**: Vite 8
- **Styling**: Tailwind CSS v4
- **Component Library**: shadcn/ui (radix-nova style)
- **Icons**: Lucide React
- **Animation**: tw-animate-css
- **Font**: Geist Variable (Fontsource)
- **Routing**: TBD
- **State Management**: TBD (currently none)

## File Structure

```
src/
├── components/
│   ├── ui/              # shadcn/ui primitives (generated)
│   ├── layout/          # Layout components (Sidebar, Header, etc.)
│   ├── shared/          # Shared domain components
│   └── pages/           # Page-level composed components
├── hooks/               # Custom React hooks
├── lib/                 # Utilities (cn, api client, etc.)
├── stores/              # State management (if needed)
├── types/               # Frontend-specific TypeScript types
├── App.tsx              # Root component with routing
├── main.tsx             # Entry point (ReactDOM.createRoot)
└── index.css            # Tailwind import + global styles
```

## Naming Conventions

| Item | Convention | Example |
|---|---|---|
| Component file | PascalCase | `UserProfile.tsx` |
| Component export | PascalCase | `export function UserProfile()` |
| Hook file | camelCase, prefixed with `use` | `useAuth.ts` |
| Hook export | camelCase, prefixed with `use` | `export function useAuth()` |
| Utility file | camelCase | `utils.ts`, `api.ts` |
| Type file | camelCase | `types.ts` |
| CSS file | Same as component | `UserProfile.css` |
| State variable | camelCase | `isLoading`, `userName` |
| Setter | prefix with `set` | `setIsLoading`, `setUserName` |

## Component Conventions

```tsx
// ✅ Correct — functional component with explicit typing
export function UserProfile({ userId, className }: UserProfileProps) {
  const { data, isLoading } = useUser(userId)

  if (isLoading) return <Skeleton className="h-20 w-full" />

  return (
    <div className={cn("flex items-center gap-3", className)}>
      <Avatar>
        <AvatarImage src={data.avatar} />
        <AvatarFallback>{data.initials}</AvatarFallback>
      </Avatar>
      <p className="text-sm font-medium">{data.name}</p>
    </div>
  )
}

interface UserProfileProps {
  userId: string
  className?: string
}
```

## Styling Rules

1. **Use Tailwind classes**, not custom CSS. Avoid `App.css`/`.module.css` unless absolutely necessary.
2. **Use `cn()` from `@/lib/utils`** for conditional className merging.
3. **Use shadcn/ui theme variables** (`--primary`, `--muted`, etc.) via Tailwind classes (`bg-primary`, `text-muted-foreground`).
4. **No inline styles** (`style={{}}`) — use Tailwind or CSS variables.
5. **Custom CSS only in `index.css`** as CSS variables or `@apply` for repeated patterns.

## shadcn/ui Usage

1. **Never manually edit** files in `src/components/ui/` — they are generated by `npx shadcn@latest add`.
2. **Add new components only when needed**: `npx shadcn@latest add <component>`.
3. **Compose** UI primitives into domain components in `src/components/shared/` or `src/components/pages/`.

## Imports

```tsx
// ✅ Correct — use @/ alias
import { Button } from "@/components/ui/button"
import { cn } from "@/lib/utils"
import { useAuth } from "@/hooks/useAuth"
import type { User } from "@/types/user"

// ❌ Wrong — avoid relative imports deep into src
import { Button } from "../../components/ui/button"
```

## State & Data Flow

1. **Keep state as local as possible** — start with `useState`, promote to context/zustand only when needed across unrelated components.
2. **Server data** should go through hooks (`useUser`, `useMessages`) that abstract fetch/cache logic.
3. **No prop drilling beyond 2 levels** — use composition or context.
